# Agent Run Lifecycle

1. Confirm the task scope and whether standing AGENTS.md rules are sufficient.
2. If not, require an AUTHORIZATION OVERRIDE block before execution.
3. Choose the closest matching policy file.
4. Create a branch and record the intended outputs.
5. Inspect the codebase before changing files.
6. Execute only within the approved scope.
7. Run targeted validation.
8. Prepare a summary with changed files, commands, risks, and rollback notes.
9. Open a PR.
10. Merge only after the required review gate is satisfied.
