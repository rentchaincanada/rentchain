Backend-only owner management APIs for delegated access.

Mission Goal

Provide landlord-owner-only APIs to view delegates, view grants, and revoke access with landlord-scoped reads, safe responses, and metadata-only audit coverage.

Scope

  • List delegates for the authenticated landlord owner.
  • List active and revoked grants for the authenticated landlord owner.
  • Revoke an active grant by grantId without deleting the grant record.
  • Resolve landlord scope only from auth context.
  • Record metadata-only audit events for revocation.

Non-goals

  • No UI or delegate management screens.
  • No email dispatch or notifications.
  • No new invitation creation or acceptance behavior.
  • No Firestore security rule changes unless absolutely required.
  • No billing delegation.
  • No property manager company hierarchy.
  • No contractor organizations.
  • No scheduling integration.

Key Principles

O
Owner-only: all endpoints require landlord owner privileges.
L
Landlord scoped: query and body params cannot override auth landlordId.
F
Fail closed: unknown grants, unknown landlords, and invalid states deny access.
S
Safe responses: do not expose tokens, hashes, or sensitive internal metadata.
A
Audit continuity: revocation creates append-safe metadata-only audit records.

API Endpoints

Method Endpoint Description
GET /api/landlord/delegated-access/delegates List delegate summaries for the landlord.
GET /api/landlord/delegated-access/grants List active and revoked grants for the landlord.
POST /api/landlord/delegated-access/grants/:grantId/revoke Revoke an active delegate grant by ID.

Acceptance Criteria

  • Owner can list delegates and grants for their landlord only.
  • Query/body landlord overrides are ignored.
  • Active grants revoke successfully.
  • Revoked or unknown grants cannot be revoked again.
  • Responses exclude token/hash and sensitive metadata.

Validation Requirements

  • Focused backend tests.
  • Backend build.
  • git diff --check.
  • No manual preview QA required unless consumed by UI.

Revocation Behavior

  • Only active grants can be revoked.
  • Revocation marks status as revoked.
  • Grant record is preserved.
  • Permission evaluator must deny revoked grants immediately.
  • Revocation stores actor, timestamp, and optional reason.

Audit Events

  • Record delegated_access_revoked.
  • Include actorUserId and actingForLandlordId.
  • Include target grant and delegate user references.
  • Include role, scope summary, reason, outcome, and timestamp.
  • Keep audit metadata-only; no tokens or sensitive payloads.

Risk Controls

  • Fail closed on ambiguous owner, landlord, grant, or scope.
  • Do not allow cross-landlord grant visibility.
  • Do not expose invitation tokens or token hashes.
  • Keep billing and settings owner-only.
  • Preserve immutable audit trail for access removal.

Merge Readiness Criteria

  • Contract review confirms owner-only scope and safe responses.
  • Focused tests cover list, scope, revoke, denied states, and audit.
  • Backend build and diff check pass.
  • No unrelated files or adjacent roadmap work included.
  • Review gate satisfied or explicit override authorized.
