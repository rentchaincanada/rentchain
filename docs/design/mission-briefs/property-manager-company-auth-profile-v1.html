Mission Brief: Property Manager Company Auth Profile V1

Add safe PM-company app profile and session support so a PM Company Owner/Admin can log in and reach company management APIs without becoming a landlord owner, without creating a landlord workspace, and without weakening existing delegate, contractor, or landlord login behavior.

1. Mission Goal

Introduce the minimum backend auth/profile foundation required for PM company users. The immediate QA target is admin+propertymanager@rentchain.ai, which has a Firebase Auth user and an active company_admin membership for Acme Property Management QA, but currently cannot safely log in because password login is landlord-biased.

2. Current Blocker

/api/auth/login calls validateLandlordCredentials(), which authenticates with Firebase and then reads or creates landlords/{uid}. PM company users must not pass through that provisioning path. Existing delegate signup avoided this by writing users/{uid} and accounts/{uid} with role: delegate, and login has an explicit delegate branch.

3. Proposed Profile Contract

Field V1 Contract
role property_manager_company as a non-landlord session role.
accountType property_manager_company for explicit profile classification.
landlordId null; never infer the user id as landlord scope for this role.
approved/status approved: true, status: active for governed QA profile setup.
Authority source Active propertyManagerCompanyMemberships, not role claims alone.

4. Implementation Scope

Login path hardening Split Firebase password validation from landlord profile loading. After Firebase auth succeeds, load users/accounts, resolve persisted safe roles first, and only run landlord provisioning for confirmed landlord users.
PM company session branch Issue a token/session with a non-landlord role, landlordId: null, empty owner permissions, and no billing/settings authority.
Governed profile setup Add a narrow dry-run-first setup path for PM company app profiles. It may create/update users/{uid} and accounts/{uid} only, with safe PM company fields.
Authorization compatibility Preserve current PM company APIs: company authority still comes from active Company Owner/Admin membership checks using the authenticated actor id.

5. Non-Goals

  • No PM Company UI changes in this mission.
  • No relationship, acceptance, or staff assignment feature changes.
  • No billing delegation, contractor organizations, tenant-facing PM UI, or custom permissions.
  • No customer data migration and no broad auth redesign beyond safe PM-company profile support.
  • No creation of landlords/{uid}, landlord workspace, landlord account, or owner billing authority for PM company users.

6. Safety Rules

  • PM company login must fail closed for missing, malformed, disabled, or ambiguous app profiles.
  • Persisted delegate and contractor login branches must remain safe.
  • Landlord login behavior must remain available for existing landlord users.
  • PM company users must not receive landlord owner permissions, billing/settings access, or landlord workspace defaults.
  • Normal setup output must not print raw IDs; safe labels only.

7. QA Profile Target

The governed setup path should support the existing Firebase Auth user admin+propertymanager@rentchain.ai and create/update a neutral PM-company app profile only after review. The existing fixture membership remains the source of company admin authority.

8. Required Tests

  • PM company admin login succeeds with a safe PM company profile.
  • PM company admin login does not create landlords/{uid}.
  • PM company session role is non-landlord and carries no landlordId.
  • PM company admin can call /api/property-manager-companies/my-companies.
  • PM company admin cannot call landlord-owner relationship routes.
  • Existing landlord, delegate, and contractor login tests still pass.
  • Unknown, missing, disabled, or malformed profiles fail closed.

9. Validation Plan

  • Focused backend auth/session tests.
  • Focused PM company route authority tests if route expectations change.
  • Backend build.
  • Frontend build only if frontend route guards or role typing are touched.
  • git diff --check.

10. Acceptance Criteria

A PM Company Admin can authenticate as a non-landlord actor, resolve company authority through active membership, and access PM company admin APIs. The same actor cannot access landlord-owner routes, and no landlord profile or workspace is created as a side effect.

11. Post-Merge Gate

After this mission merges, use the governed setup path to create the safe app profile for admin+propertymanager@rentchain.ai, confirm login, then resume #1229 manual QA. #1229 remains draft until that QA passes.
