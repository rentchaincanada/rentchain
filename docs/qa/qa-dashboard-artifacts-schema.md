# QA Dashboard Artifact Index Schema

## Overview

`qa-dashboard-index.json` is the Phase 0A dashboard artifact index generated by the Playwright QA artifact reporter. It is written beside `qa-report.json` in the configured artifact directory:

```text
test-results/qa-artifacts/qa-dashboard-index.json
```

The index aggregates existing QA report data. It does not create new test coverage, modify application data, or call backend services.

## Root Object

```ts
type QADashboardIndex = {
  schemaVersion: "1.0.0";
  metadata: DashboardMetadata;
  phaseCompletion: PhaseCompletion;
  coverage: CoverageSummary;
  mobileBaseline: MobileBaselineSummary;
  artifactInventory: ArtifactInventoryItem[];
  sanitizationAudit: ArtifactSanitizationAudit;
  signOffChecklist: PhaseSignOffChecklist;
};
```

The schema version is fixed at `1.0.0` for Phase 0A.

## Metadata

```ts
type DashboardMetadata = {
  generatedAt: string;
  branch?: string;
  commit?: string;
  environment: "local" | "ci" | "unknown";
  sourceReportSchemaVersion: string;
};
```

Rules:

- `generatedAt` is an ISO timestamp.
- `branch` is sanitized and length-limited.
- `commit` is limited to a short commit prefix.
- `environment` is derived from the current runtime.
- `sourceReportSchemaVersion` mirrors the input `qa-report.json` schema version.

## Phase Completion

```ts
type PhaseCompletion = {
  phase: "Phase 0A";
  status: "complete" | "blocked" | "manual-review-required";
  completedPullRequests: string[];
  sourceOfTruth: string;
};
```

`completedPullRequests` lists the Phase 0A merge sequence represented by the capstone. `sourceOfTruth` points to `.handoff/merge-log.md`.

## Coverage Summary

```ts
type CoverageSummary = {
  roles: Record<"tenant" | "landlord" | "admin", RoleCoverageBaseline>;
  aggregate: {
    totalTests: number;
    passedTests: number;
    failedTests: number;
    skippedTests: number;
    totalSmokeRoutesTested: number;
    totalMobileCombinationsTested: number;
  };
};
```

Each role baseline includes:

```ts
type RoleCoverageBaseline = {
  role: "tenant" | "landlord" | "admin";
  smokeRoutesTested: number;
  smokeRoutesWithBlockers: number;
  mobileCombinationsTested: number;
  mobileCombinationsWithFailures: number;
  knownMobileBlockers: number;
  endpointCoverageEstimate: number;
  testStats: {
    total: number;
    passed: number;
    failed: number;
    skipped: number;
  };
};
```

The dashboard does not infer hidden coverage. It summarizes what the current QA report emitted.

## Mobile Baseline

```ts
type MobileBaselineSummary = {
  totalCombinationsTested: number;
  totalCombinationsWithFailures: number;
  knownBlockers: Array<{
    role: "tenant" | "landlord" | "admin";
    route: string;
    viewport: string;
    issue: string;
    category: string;
  }>;
};
```

Known blockers are carried forward from existing mobile layout baseline data. The capstone does not reclassify new layout failures into the accepted baseline.

## Artifact Inventory

```ts
type ArtifactInventoryItem = {
  path: string;
  kind: "qa-report" | "qa-dashboard-index" | "playwright-json" | "unknown-json";
  bytes: number;
  schemaVersion?: string;
  validationStatus: "pass" | "fail" | "not-applicable";
};
```

Inventory entries are relative to the `qa-artifacts` directory. The index validates `qa-report.json` using the existing QA report validator.

## Sanitization Audit

```ts
type ArtifactSanitizationAudit = {
  status: "pass" | "fail";
  artifactsScanned: number;
  jsonValuesScanned: number;
  findings: SanitizationFinding[];
  scannedAt: string;
};
```

Findings use this structure:

```ts
type SanitizationFinding = {
  artifactPath: string;
  jsonPath: string;
  pattern: string;
  severity: "warning" | "failure";
  message: string;
};
```

The audit intentionally omits matched values. It reports where and why a value was flagged without copying secrets, tokens, storage paths, or provider payloads into the dashboard index.

The audit checks for:

- secret key style strings
- bearer tokens
- JWT-like values
- storage paths
- long opaque token-like values
- UUID-like identifiers
- invalid JSON

Deterministic smoke fixture labels and explicitly allowed test fixture emails are permitted.

## Sign-Off Checklist

```ts
type PhaseSignOffChecklist = {
  phase: "Phase 0A";
  status: "complete" | "blocked" | "manual-review-required";
  gates: PhaseSignOffGate[];
  operatorSignOffRequired: true;
  immutableAfterSignOff: true;
};
```

Each gate is:

```ts
type PhaseSignOffGate = {
  id: string;
  label: string;
  status: "pass" | "fail" | "manual-review-required";
  evidence: string[];
  manualReviewReason?: string;
};
```

`manual-review-required` means an operator must inspect or authorize the decision. It is not an automatic failure.

## Validation Rules

The dashboard helper exports:

- `generateDashboardIndex(report, options)`
- `writeDashboardIndexArtifact(report, outputDir)`
- `validateQADashboardIndex(index)`
- `evaluatePhase0ASignOffChecklist(report, audit, inventory)`

Validation confirms the root structure, schema version, required metadata, coverage sections, artifact inventory, sanitization audit, and sign-off checklist shape.

## Consumption Rules

Downstream tooling may read `qa-dashboard-index.json` as an evidence artifact, but must not treat it as automatic release approval.

Consumers must preserve these rules:

- Do not upload storage-state JSON, cookies, tokens, screenshots with secrets, videos, traces, or downloaded private data.
- Do not treat warnings as accepted without operator review.
- Do not infer role coverage beyond the run represented by the input QA report.
- Do not mutate application data based on dashboard index contents.
- Do not start future phase automation without a new mission.

## Troubleshooting

If `qa-dashboard-index.json` is missing:

1. Confirm the Playwright reporter is configured.
2. Confirm `qa-artifact-reporter.ts` ran at test completion.
3. Confirm the configured output directory is writable.

If schema validation fails:

1. Open `qa-report.json`.
2. Confirm `schemaVersion` is `1.0.0`.
3. Confirm `smokeFindings` includes tenant, landlord, and admin keys.
4. Confirm `mobileLayoutRegression` includes tenant, landlord, and admin keys.

If sanitization fails:

1. Review finding paths and pattern names.
2. Do not copy matched values into tickets or PRs.
3. Remove or redact the source artifact.
4. Regenerate the artifact after the source issue is resolved.

If a gate is `manual-review-required`:

1. Read the gate evidence and reason.
2. Decide whether the state is expected for the current run.
3. Record the operator decision outside the generated JSON artifact.
