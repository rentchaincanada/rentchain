# Consent and Retention Checklist

## Consent Checklist

- [ ] Consent capture text is explicit and jurisdiction-appropriate.
- [ ] Consent timestamp is stored.
- [ ] Consent actor (who captured/approved) is stored.
- [ ] Consent status is validated before provider request execution.
- [ ] Consent withdrawal behavior is defined.

## Retention Checklist

- [ ] Screening records have a documented retention window.
- [ ] Audit records have a documented retention window.
- [ ] Deletion and suppression paths are documented.
- [ ] Access to retained records is role-scoped.
- [ ] Export/disclosure handling is documented for legal requests.

## Operational Checklist

- [ ] Correlation identifier present for provider failures.
- [ ] Alerting path defined for repeated provider failures.
- [ ] Provider downtime fallback behavior documented.
- [ ] Incident response ownership is assigned.
